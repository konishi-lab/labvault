# labvault 進捗報告 (2026-03 〜 2026-06)

**Python/Notebook で実験する研究室のための実験データ基盤**
測定 → 解析までのコード・データ・条件を自動記録し、蓄積データを LLM が横断検索・解析する。
先行プロジェクト **arim-mdxdb** を引き継ぐ形で 2026-03 に再出発。ARIM MDX 上の既存ストレージ資産はそのまま流用しつつ、設計と SDK を一から作り直している。

## 3. 技術的ハイライト

- **実験の自動記録**: Jupyter Notebook で実行したセルのコード・変数の変化・エラー情報を、人手の操作なしで実験単位 (record) として保存。装置制御用の Python スクリプトでも 1 行 (`exp.log_value(...)`) 書くだけで条件・結果が残る。
- **低コスト・低運用なクラウド構成**: 既存の研究室インフラ (ARIM MDX) を巨大ファイル置き場としてそのまま流用し、検索用メタデータだけを Google Cloud の無料枠に載せる構成。月額 $1 以下で運用。
- **AI アシスタントからの直接アクセス**: Claude / Gemini など MCP に対応した AI アシスタントから自然言語で「ある条件範囲の実験を抽出して比較したい」と頼むと、データベース問い合わせ・統計集計・ファイルプレビューが裏側で自動連鎖。実験者は SQL や Python を書かなくて良い。
- **Web 画面**: ブラウザでレコード一覧、条件カラム表示、条件 vs 結果の散布図 (2026-04 〜)、NxM グリッドでの一括アップロード、研究室間の共有モーダル、Notebook 実行ログ閲覧。
- **研究室を跨いだ共有 (6 月後半)**: 別研究室メンバーに「閲覧のみ / 解析投稿可」の 2 段階で record を共有。公開リンク方式にも対応し、URL に乗せたトークンがサーバーログに残らない設計でセキュリティも担保。

## 4. 今後 3 か月の方針

1. **装置データの自動取り込み** — テンプレートに紐付けて `.ras` (XRD) などの装置ファイルから条件を自動抽出。実際に必要な装置が確定してから順に対応。
2. **解析の系譜の記録** — 「ある解析がどの実験データから出てきたか」を再実行できる形で保存し、後から検証 / 追試できるように。
3. **実利用の定着** — 実際の実験者に使ってもらい、装置ごとの記録手順の文書化とフィードバックの吸い上げ。
